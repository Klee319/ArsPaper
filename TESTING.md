# ArsPaper 網羅的動作確認手順書

## 前提条件

- Paper 1.21.4+ サーバーが起動済み
- `build/libs/ArsPaper-<version>.jar` を `plugins/` に配置しサーバー再起動済み
- OP権限を持つプレイヤーでログイン
- テスト用ワールドはクリエイティブモード推奨（素材確保のため）
- **サバイバルモードで確認が必要な項目には `[SV]` マークを付記**

---

## 1. プラグイン起動・設定ファイル確認

### 1.1 起動確認
- [ ] サーバーログに `ArsPaper enabled!` が表示される
- [ ] エラーやWarningが出ていないことを確認
- [ ] `plugins/ArsPaper/config.yml` が生成される
- [ ] `plugins/ArsPaper/recipes.yml` が生成される
- [ ] `plugins/ArsPaper/rituals.yml` が生成される
- [ ] サーバーログに `Loaded N recipes from recipes.yml` が表示される

### 1.2 設定ファイル確認
- [ ] `config.yml` のデフォルト値確認:
  - `default-max: 100`
  - `default-regen-rate: 2`
  - `regen-interval-ticks: 20`
  - `per-glyph-unlock-bonus: 5`
- [ ] `recipes.yml` に全22レシピが含まれる（魔法書3種、杖1種、ブロック4種、台座、儀式の核、防具12種）
- [ ] `rituals.yml` に6種の儀式レシピが含まれる

## 2. マナシステム

### 2.1 BossBar表示
- [ ] ログイン後にBossBarが表示される（「マナ: 100 / 100」）
- [ ] `/ars mana` で「マナ: 100 / 100」のようなメッセージが表示される

### 2.2 マナ回復
**手順:** サバイバルモードに切替 → スペルを発動してマナ消費 → 回復を観察
- [ ] マナを消費後、約1秒ごとに2ずつ回復することを確認
- [ ] BossBarの値がリアルタイムで更新される
- [ ] 最大マナまで回復したら回復が停止する

### 2.3 再ログイン永続性
- [ ] ログアウト→ログインでマナ値が保持される
- [ ] BossBarが再表示される
- [ ] `/ars mana` で保持された値が表示される

## 6. スペル作成GUI（SpellCraftingGui）

### 6.1 GUI構造確認
1. スペルブックを手に持ってスニーク+左クリック
- [ ] 6段のGUIが開く
- [ ] 上段: 黒ガラスパネルの枠内にスペル構成エリア（最大8スロット）
- [ ] 中段: グリフパレット（利用可能なグリフ一覧）
- [ ] 下段: 操作ボタン（形態/効果/増強タブ、全消去、元に戻す、スペル保存）

### 6.2 タブ切替
- [ ] 「形態」ボタンクリック → Form一覧表示（Projectile, Touch, Self, Underfoot）
- [ ] 「効果」ボタンクリック → Effect一覧表示（Break, Harm, Heal, Grow, Light, Speed, Blink, Bounce）
- [ ] 「増強」ボタンクリック → Augment一覧表示（Amplify, AOE, ExtendTime, Pierce）

### 6.3 グリフ追加・除去
**前提:** 筆記台で少なくとも「Projectile」「Harm」「Amplify」をアンロック済みであること

1. 「形態」タブでProjectileクリック
   - [ ] 上段の構成エリアに追加される（種類: 形態、マナ: X、クリックで除去 と表示）
2. 「効果」タブに切替 → Harmクリック
   - [ ] 2番目に追加される
3. 構成エリアの1番目（Projectile）をクリック
   - [ ] 除去される

### 6.4 操作ボタン
- [ ] 「元に戻す」ボタン → 最後に追加したグリフが除去
- [ ] 「全消去」ボタン → 全グリフが除去
- [ ] 空の状態で「スペル保存」→ 「スペルが空です！」エラー

### 6.5 バリデーション
- [ ] 最初のグリフとしてEffectを追加 → 「最初のグリフは形態(Form)である必要があります！」
- [ ] 8グリフ目を超えて追加 → 「スペルが満杯です！（最大8グリフ）」
- [ ] 未アンロックグリフをクリック → 「このグリフは未解放です！」
- [ ] ティア制限超過のグリフ（例: 初級ブックでBlink）→ 「このスペルブックはティアX以下のグリフのみ使用可能です！」

### 6.6 スペル保存
**手順:** Projectile → Harm の順で構成を組む
1. 「スペル保存」ボタンクリック
   - [ ] 「スロット1に保存しました！（コスト: Xマナ）」メッセージ
   - [ ] GUIが自動で閉じる
2. 再度スニーク+左クリックでGUIを開く
   - [ ] 保存した構成（Projectile + Harm）が上段に表示されている

## 7. スペル発動テスト

### 7.1 基本発動 [SV]
**前提:** スペルブックにスペルを設定済みであること

1. スペルブックを手に持つ
2. 右クリック
   - [ ] スペルが発動する
   - [ ] マナが正しく減少する（BossBar更新）
   - [ ] 0.5秒以内に再クリック → 発動しない（クールダウン）
   - [ ] マナ不足時 → 「マナが不足しています！」メッセージ

### 7.2 各Form動作テスト

#### Projectile（投射）
**構成:** `Projectile + Harm`
- [ ] 右クリックでSnowball（雪玉）が発射される
- [ ] 壁に当たると消滅
- [ ] エンティティ（Mob/プレイヤー）に当たるとダメージを与える

#### Touch（接触）
**構成:** `Touch + Break`
- [ ] 5ブロック以内のブロックを見て右クリック → そのブロックが破壊される
- [ ] 5ブロック以上離れている場合 → 効果なし

#### Self（自己）
**構成:** `Self + Speed`
- [ ] 右クリック → 自分に移動速度上昇ポーション効果が付与される

#### Underfoot（足元）
**構成:** `Underfoot + Grow`（畑の上に立って）
- [ ] 右クリック → 足元の作物が成長する

### 7.3 各Effect詳細テスト

| 構成 | テスト手順 | 確認項目 |
|------|-----------|---------|
| Self + Harm | サバイバルで自分に使用 | [ ] 自分にダメージ |
| Self + Heal | ダメージを受けた状態で使用 | [ ] HPが回復する |
| Touch + Break | ブロックを見て使用 | [ ] 対象ブロックが破壊 |
| Underfoot + Grow | 畑の上で使用 | [ ] 足元の作物が成長 |
| Self + Light | 使用 | [ ] 暗視＋発光が付与 |
| Touch + Light | 空気ブロックを見て使用 | [ ] 光源（LIGHT）が設置 |
| Self + Speed | 使用 | [ ] 移動速度上昇付与 |
| Self + Blink | 方向を向いて使用 | [ ] その方向にテレポート |
| Projectile + Bounce | Mobに命中させる | [ ] 対象が打ち上げられる |

### 7.4 Augmentテスト

| 構成 | テスト手順 | 確認項目 |
|------|-----------|---------|
| Projectile + Harm + Amplify | Mobに命中 | [ ] 通常Harmより大きなダメージ |
| Touch + Grow + AOE | 畑を見て使用 | [ ] 周囲3ブロック範囲の作物も成長 |
| Self + Speed + ExtendTime | 使用 | [ ] 通常より長い持続時間（+200tick≒10秒延長） |
| Projectile + Harm + Pierce | 2体並んだMobに命中 | [ ] 1体目を貫通して2体目にもダメージ |
| Self + Blink + Amplify | 使用 | [ ] テレポート距離が増加（通常8→12ブロック） |
| Self + Speed + Amplify + Amplify | 使用 | [ ] 速度上昇のレベルが上がる |

### 7.5 BlinkEffect安全性テスト
- [ ] 壁に向かってBlink → 壁の手前にテレポート（めり込まない）
- [ ] 空中に向かってBlink → 安全な着地点に降ろされる
- [ ] 虚空（Y < 0）方向にBlink → テレポートしない（安全な場所なし）
- [ ] Blink先が溶岩/水中でない安全な地面であること

### 7.6 BounceEffect制限テスト
- [ ] Bounce + Amplify × 多数でも打ち上げ速度が上限4.0を超えない

## 8. 筆記台（Scribing Table）- グリフアンロック

### 8.1 設置とGUI
1. `/ars give scribing_table` で筆記台を入手
2. 地面に設置
   - [ ] ArmorStandによるカスタム見た目が表示される
3. 右クリック
   - [ ] 「筆記台」タイトルのGUIが開く
   - [ ] 全グリフが一覧表示される（未アンロック=石炭アイコン、アンロック済=素材アイコン）

### 8.2 アンロックコスト確認
各グリフにカーソルを合わせてLoreを確認:
- [ ] Tier 1グリフ: 「必要: 5レベル + ラピスラズリ1個」
- [ ] Tier 2グリフ: 「必要: 10レベル + ラピスラズリ4個」
- [ ] Tier 3グリフ: 「必要: 20レベル + ラピスラズリ8個」
- [ ] 種類表示: 「種類: 形態/効果/増強」
- [ ] マナコスト: 「マナコスト: X」
- [ ] ティア表示: 「ティア: X」

### 8.3 アンロック実行 [SV]
**手順:**
1. サバイバルモードに切替
2. `/xp set @s 30 levels` でレベルを確保
3. ラピスラズリをインベントリに用意
4. 筆記台GUIを開く
5. 未アンロックグリフ（例: Projectile）をクリック

確認:
- [ ] レベルが消費される（5レベル減少）
- [ ] ラピスラズリが消費される（1個減少）
- [ ] 「解放: Projectile！ (最大マナ+5)」メッセージ
- [ ] `/ars mana` でマナ上限が+5されている
- [ ] GUIが再描画され、アンロック済（ダイヤアイコン、「[解放済]」表示）になる

### 8.4 アンロック失敗ケース
- [ ] レベル不足でクリック → 「レベルが不足しています！必要: X」
- [ ] ラピスラズリ不足でクリック → 「ラピスラズリが不足しています！必要: X個」
- [ ] 既にアンロック済みをクリック → 「すでに解放済みです！」
